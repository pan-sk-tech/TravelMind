## GitHub 提交和本地保留的分界

可以提交：

- `training/docs/**`
- `training/configs/**` 中的主线配置
- `training/scripts/**` 中当前还会运行的脚本
- `training/data/planner/eval/`、`training/data/planner/eval_hard/`
- `training/data/llamafactory/dataset_info.json` 和 `manifests/`
- `training/outputs/eval/reports/` 中整理好的轻量报告

只留本地：

- 模型权重、checkpoint、merged model
- `generations.jsonl`、serve log、长训练日志
- `data/planner/dpo/`、`bestofn/`、`sft_realbudget_runs/` 和 `llamafactory/generated/`
- `private-docs/`、`.hf_cache/`、`data/cache/`
- 本地助手 skill：`skills/trip-planner-model-eval/`

## 如果以后要清空间

优先级从高到低：

1. 清 `.hf_cache/`、`data/cache/`、`__pycache__/`、旧 serve log。
2. 清 `outputs/qwen3_8b/` 这类非主线探索。
3. 把 `training/archive/` 和 `outputs/archive/` 打包到外部盘，再从工作区删。
4. 对旧 SFT/DPO adapter，只保留被文档引用的关键 checkpoint 和 `trainer_log.jsonl`。
5. 不要直接删四个 merged model，除非已经确认不再需要继续训练或评估。
