# DPO训练实验记录卡：260518 core1418 plus direct402

本文档用于记录本轮 DPO 训练的训练数据配比、训练参数、训练日志观察口径和后续评估重点。它的定位不是最终评测报告，而是实验过程记录，方便之后解释“为什么这轮训练有效或无效”。

## 一、为什么训练日志和数据配比很重要

DPO 的最终效果不能只看最后一个 checkpoint 的评测分数。训练日志、数据配比和训练参数共同决定了这轮实验的可解释性：

- 训练日志告诉我们模型是不是稳定地学到了 chosen/rejected 的偏好差异。
- 数据配比告诉我们模型主要被哪些错误信号驱动。
- 训练参数告诉我们这轮更新强度有多大，是否存在过训、欠训或显存策略变化带来的差异。
- checkpoint sweep 告诉我们最佳点出现在第几轮，而不是盲目相信 final。

如果只保留最终 eval，后面看到 planner soft 上升或下降时，很难判断是数据质量、数据分布、学习率、epoch 数、checkpoint 选择，还是评估噪声造成的。

## 二、本轮训练目标

本轮训练目标是继续优化 planner soft pass。

当前判断是：planner soft 比旧 soft 指标更贴近旅行规划质量，因为它关注的是旅行计划是否在结构、预算、餐饮尺度、grounding 和整体体验上都可用，而不是只满足若干机械规则。DPO 数据也围绕 planner soft 的失败模式构造和筛选。

本轮设置了：

```bash

```

该优化用于降低 DPO 长上下文训练的显存占用。当前普通 sigmoid DPO 下，它不应改变 DPO 数学目标，只是减少无关 prompt 位置的 logits 物化。

## 六、当前日志观察

截至 step 89/140，训练状态如下：

- 进度：约 64%
- epoch：约 2.55
- 最近 loss 区间：约 0.02 到 0.22，有明显震荡
- reward accuracy：大多在 0.94 到 1.0
- reward margin：大多在 4.0 以上
- grad norm：约 0.5 到 3.8，没有爆炸

对 DPO 来说，单 step loss 震荡是正常现象。更重要的是看下面几个趋势：

- `rewards/accuracies` 是否长期维持高位。
- `rewards/margins` 是否持续为正且没有塌到 0 附近。
- `grad_norm` 是否出现连续异常升高。
- rejected reward 是否持续低于 chosen reward。

当前从日志看，loss 虽然抖，但偏好区分仍然健康。

## 八、后续复盘要回答的问题

评估结束后，需要把训练日志和结果一起看：

- planner soft 是否继续提升。
- hard pass 有没有被牺牲。
- 预算、餐饮尺度、grounding、结构类错误分别有没有改善。
- 最佳 checkpoint 出现在第几个 epoch。
- 如果 final 低于中间 checkpoint，说明 4 epoch 可能过多。
- 如果 checkpoint-18 到 checkpoint-126 基本单调提升，说明数据方向稳定，可以继续扩数据。
- 如果 reward margin 很高但 eval 不涨，说明训练偏好信号和真实 planner soft 可能不完全一致，需要重新审数据。

## 九、推荐的固定记录口径

以后每轮 DPO 都建议固定记录：

1. 数据配比：每个来源多少条、错误类型分布、reject 是单错误还是多错误。
2. 数据质量：去重、截断率、chosen/rejected 长度分布、明显脏样本比例。
3. 训练参数：base、adapter 起点、LR、beta、epoch、batch、cutoff、offload、label logits 优化。
4. 训练日志：loss、reward accuracy、reward margin、grad norm、每 step 速度。
5. checkpoint sweep：每 0.5 epoch 的 planner soft、hard pass、关键子指标。
6. 结论：最佳 checkpoint、是否继续训练、下一批数据应该补哪类失败模式。
