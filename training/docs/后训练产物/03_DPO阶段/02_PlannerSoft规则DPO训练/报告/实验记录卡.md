# DPO训练实验记录卡：260518 fd600 planner soft direct402

本文档记录 2026-05-18 这轮 `fd600 planner_soft direct402` DPO 训练。它是当前 `core1418_plus_direct402_from_ckpt25` 的上一轮，训练结果里 `checkpoint-25` 被选为后续续训起点。

## 一、这轮训练要解决什么

这轮之前，我们已经发现旧 `dpo_soft_pass` 和旅行质量之间有一定错位：一些预算字段看起来过了，但按实际酒店、餐饮、景点重新计算并不一定合理。因此这轮把目标转向更贴近最终体验的 `planner_soft_pass`。

这一轮的核心意图是：用更明确的 planner soft 失败信号做 DPO，让模型少犯预算一致性、餐饮价格尺度、行程结构和 grounding 相关错误。

## 七、主要结论

1. DPO 对 planner soft 明显有效。`ckpt-25` 的 planner soft 从 SFT final 的 35.2% 提升到 44.7%，提升 9.5pp。
2. 餐饮价格尺度修正非常强。MealScale 从 SFT final 的 81.1% 提升到接近 98%。
3. final 不等于最佳。`ckpt-35` 开始回落，说明 4 epoch 并不一定都有效，checkpoint sweep 是必要的。
4. `ckpt-25` 综合最好，因此被选为下一轮 `core1418_plus_direct402` 的续训起点。
5. 这轮暴露了一个工程坑：长上下文 DPO 如果不做 label logits 优化，很容易在 40G 卡上 OOM。

## 八、给下一轮的启发

- 数据方向是对的，可以扩大 planner soft 数据量。
- 不应该只保留 final，必须每 0.5 epoch 保存可评测 adapter。
- 后续数据要继续控制 rejected 错误数量，尽量避免多错误混在一起让信号变脏。
- 训练日志要和 checkpoint sweep 一起看：loss 下降不是目的，planner soft 和 hard pass 才是目的。
