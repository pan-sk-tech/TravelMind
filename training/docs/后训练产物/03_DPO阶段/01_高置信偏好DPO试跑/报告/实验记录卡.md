# DPO训练实验记录卡：260517 high confidence strict

本文档记录 2026-05-17 这轮 `high_conf_strict` DPO。它是 SFT final 之后的第一轮主线 DPO，也就是后续 `direct402` 和 `core1418_plus_direct402` 的上上轮。

## 一、这轮训练要解决什么

这轮 DPO 的目标是验证：在 SFT 已经把 schema、日期、grounding 等基础能力做稳后，能否用 chosen/rejected 偏好样本进一步改善预算、餐饮尺度、结构和整体体验。

当时我们还没有完全确定 planner soft 是主目标，所以数据分布更像“高置信偏好样本的综合压力测试”，覆盖多个错误桶，而不是只围绕 planner soft 做直接筛选。

## 七、主要结论

1. 这轮验证了 DPO 是有效方向：hard pass、预算算术、预算关系、餐饮尺度、hard 集 recomputed soft 都有改善。
2. 旧 `dpo_soft_pass` 出现回退，说明旧 soft composite 和真实旅行质量存在错位。
3. final step 指标回落，但没有中间 checkpoint 可评估，这是训练流程上的主要问题。
4. 数据分布偏综合，reject 可能同时带多个问题，信号没有后续 direct402 那么干净。
5. 这轮直接促成了后面两件事：重新定义 planner soft 目标，以及每 0.5 epoch 保存可评测 adapter。

## 八、给后续的启发

- 第一轮 DPO 不应该只保存 final。
- 训练日志里的 reward accuracy / margin 必须和最终 eval 一起看。
- 如果多个可解释子指标提升但 soft pass 下降，要怀疑 soft 指标本身是否错配。
- 下一批数据应该更干净，chosen/rejected 的目标差异要更直接。
- 数据配比需要记录，否则后面很难解释模型到底学到了哪类偏好。
