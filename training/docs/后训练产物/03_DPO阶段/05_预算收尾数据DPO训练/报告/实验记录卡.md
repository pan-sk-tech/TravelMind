# 实验记录卡：260520 closing budget clean from ckpt138

## 目标

`260519 checkpoint-138` 已经把单生成 planner soft 拉高，但 bad case 仍集中在预算和重复上：有些计划不超预算却过于保守，有些酒店、景点、餐饮细项预算关系不稳，hard split 上还会出现餐饮或景点重复。

这一炉的目标不是继续堆泛化数据，而是构造一批干净的预算专项 pair，集中修这些收尾问题。

## 数据构造

数据摘要：`training/data/planner/dpo/260520_closing_budget_clean/summary.json`

这批数据不从评测集挖答案，而是从已有训练候选池筛选预算相关 clean pair。关键 anti-leak 检查如下：

数据质量约束：

- chosen 必须 `schema_ok && generation_clean && sft_hard_pass && planner_soft_pass`。

- one pair per request signature。
- 冻结 eval 请求签名在 pair selection 前跳过。
- 人工 review 样例见 `training/data/planner/dpo/260520_closing_budget_clean/review_samples.md`。

## 结论

这炉的数据是干净的，尤其 `selected_eval_signature_overlap = 0` 很关键。但它没有成为更好的单生成版本：hard pass 稍涨，planner soft 和重算预算 soft 回落。

阶段判断是：这批 closing 数据可以改变候选分布，但单次生成收益不足，不能替代 `260519 checkpoint-138` 作为单生成主推点。
