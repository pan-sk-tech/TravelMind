# 评估记录：260521 ckpt64 vs 260519 ckpt138 rerank n4

## 目标

`260521 ckpt64` 单生成不如 `260519 ckpt138`，但它可能在多温度采样下提供更好的候选分布。因此最后做 `ckpt64` 和 `ckpt138` 的 rerank n4 对比，判断展示版本该选哪个。

## 推理设置

评估输出：`training/outputs/eval/by_model/dpo_260521_ckpt64_260519_ckpt138_rerank_n4_gpu0_2_w8/`

对比报告：`training/outputs/eval/by_model/dpo_260521_ckpt64_260519_ckpt138_rerank_n4_gpu0_2_w8/ckpt64_vs_ckpt138_rerank_n4_report/ckpt64_vs_ckpt138_rerank_n4_full_report.md`

设置：

- 每条 prompt 使用 4 个候选：旧 `t02` + 新生成 `t04/t06/t08`。
- standard 200 条，hard 300 条。
- 两个模型使用同一批 records。
- 最终输出由 backend rerank 选择。

## 和 MiMo 外部强模型的关系

历史上跑过 `mimo_v2_5_pro_external_mt1p5`，它是 MiMo v2.5 Pro 外部 API，w50，max token 按 1.5x 放大。这个结果来自较早规则和路径，不能当严格同版本横评，但可以作为强模型参照。

| 模型 | planner soft | recomputed budget soft | hard pass | 备注 |
| --- | ---: | ---: | ---: | --- |
| MiMo v2.5 Pro mt1p5 | 78.7% | 76.6% | 98.8% | 预算总数贴合强，但预算关系和餐饮尺度不完全符合本地规则。 |
| ckpt64_rerank_n4 | 80.6% | 68.2% | 99.4% | planner soft 和 hard pass 更高，重算预算 soft 仍低于 MiMo。 |

对外表述要克制：当前本地模型在本项目规则口径下 planner soft 已经追上强模型参照，但预算贴合还有继续优化空间。

## 结论

如果只做单生成，`260519 ckpt138` 更稳；如果允许多生成 rerank，`ckpt64_rerank_n4` 是当前最终展示版本。它在 hard split 上提升更明显，符合 closing budget 数据想补的方向。

最终展示建议写成：`ckpt64_rerank_n4`，500 条 planner soft `80.6%`，hard split planner soft `77.0%`。
