# 实验记录卡：DPO 多候选 Rerank 最终选择

## 一、阶段目标

这一阶段用于最终展示版本选择。DPO 收尾后，`260519 ckpt138` 是单生成最稳版本，而 `260521 ckpt64` 单生成不占优，但它在多温度采样下可能提供更好的候选分布。因此需要在同一批评测记录上比较两者的 rerank n4 结果。

## 二、推理设置

详细评估：`报告/评估记录.md`

| 项 | 内容 |
| --- | --- |
| 对比模型 | `260521 ckpt64` vs `260519 ckpt138` |
| 候选数 | 每条 prompt 4 个候选，旧 `t02` + 新 `t04/t06/t08` |
| 评估集 | standard 200 条 + hard 300 条 |
| 选择方式 | backend rule/rerank 选择最终输出 |
| 输出目录 | `training/outputs/eval/by_model/dpo_260521_ckpt64_260519_ckpt138_rerank_n4_gpu0_2_w8/` |

## 三、500条合并结果

| 指标 | ckpt64_rerank_n4 | ckpt138_rerank_n4 | 变化 |
| --- | ---: | ---: | ---: |
| hardpass | 99.4% | 98.2% | +1.2pp |
| planner soft | 80.6% | 79.0% | +1.6pp |
| recomputed budget soft | 68.2% | 65.4% | +2.8pp |
| budget preference aligned | 86.0% | 85.6% | +0.4pp |
| recomputed budget fit | 73.4% | 71.0% | +2.4pp |

## 四、分 split 观察

| split | 指标 | ckpt64_rerank_n4 | ckpt138_rerank_n4 | 变化 |
| --- | --- | ---: | ---: | ---: |
| standard | planner soft | 86.0% | 87.0% | -1.0pp |
| standard | recomputed budget soft | 79.0% | 77.0% | +2.0pp |
| hard | planner soft | 77.0% | 73.7% | +3.3pp |
| hard | recomputed budget soft | 61.0% | 57.7% | +3.3pp |
| hard | hardpass | 99.3% | 97.7% | +1.6pp |

## 五、阶段结论

1. 如果只做单生成，优先选择 `260519 ckpt138`。
2. 如果允许多生成 rerank，最终展示版本选择 `ckpt64_rerank_n4`。
3. `ckpt64_rerank_n4` 的主要优势在 hard split，更符合 closing budget 数据想补的方向。
4. 对外展示要写清楚这是 rerank n4 结果，不应和单生成 checkpoint 混成一个口径。
