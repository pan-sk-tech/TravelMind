# 实验记录卡：SFT 多温度候选 Rerank

## 一、阶段目标

这一阶段验证一个推理侧假设：模型单次生成不稳定时，多温度采样得到的候选池里是否已经包含更好的旅行计划。如果有，再用规则评测或 rerank 选择最终答案，可能比继续盲目训练更稳。

这批实验面向 SFT checkpoint，比较 `ckpt104`、`final1200`、`old600final` 在 standard / hard 评估集上的 rerank 结果。

## 二、推理与归档口径

| 项 | 内容 |
| --- | --- |
| 候选来源 | SFT checkpoint 多温度生成。 |
| 选择方式 | 规则指标 / rerank merge。 |
| standard records | `training/data/v3/eval/records.jsonl`，200 条。 |
| hard records | `training/data/v3/eval_hard/records.jsonl`，300 条。 |
| 脚本归档 | `配置/` 下的 generate、rule metrics、pipeline、slice report、rerank 脚本。 |

## 三、关键结果

standard split：

| 模型 | hardpass | softpass | 重算预算 softpass | 重算预算贴合 |
| --- | ---: | ---: | ---: | ---: |
| ckpt104 rerank | 98.50% | 73.00% | 64.00% | 71.50% |
| final1200 rerank | 98.00% | 76.00% | 69.00% | 74.50% |
| old600final rerank | 99.50% | 76.50% | 71.50% | 77.00% |

hard split：

| 模型 | hardpass | softpass | 重算预算 softpass | 重算预算贴合 |
| --- | ---: | ---: | ---: | ---: |
| ckpt104 rerank | 97.67% | 60.67% | 48.33% | 57.67% |
| final1200 rerank | 98.33% | 60.67% | 45.00% | 57.33% |
| old600final rerank | 97.33% | 59.33% | 51.00% | 61.67% |

## 四、阶段结论

1. 多候选 rerank 能把 SFT 模型的 hardpass 稳定推到 97% 以上。
2. standard split 上 `old600final` 的预算贴合和 softpass 最好。
3. hard split 上三个版本非常接近，`old600final` 的重算预算贴合略好，`final1200` 的 hardpass 略高。
4. Rerank 证明了候选池里有可捞的好答案，但规则目标会决定它捞出哪类答案。
5. 这一步为后续 DPO rerank 提供了推理框架和报告模板。
