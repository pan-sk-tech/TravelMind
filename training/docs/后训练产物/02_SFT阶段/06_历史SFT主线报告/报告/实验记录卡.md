# 实验记录卡：历史 SFT 主线报告

## 一、阶段目标

这个目录归档早期 SFT 与 Base 的阶段评估。它不是最终 SFT 主线，而是解释一个重要转向：SFT 能学会 TripPlan 结构和一部分硬规则，但单靠 SFT 并没有稳定提升复杂决策质量，尤其预算偏好会退化。

| 指标 | Base + prompt v3 | SFT + prompt v3 | 判断 |
| --- | ---: | ---: | --- |
| JSON 提取成功率 | 100.00% | 100.00% | 持平 |

| 平均延迟 | 41.18s | 51.99s | SFT 更慢 |

## 四、阶段判断

SFT 学到了：

- TripPlan JSON 和 schema 结构。
- 每天景点数量控制。
- 天气、住宿、餐饮完整性等部分硬协议。
- 更像 teacher 的输出风格。

SFT 没学稳：

- 预算偏好和预算档位选择。
- 预算分项加总能力。
- 高预算用户应获得更高品质选择的偏好判断。
- 推理速度和输出长度控制。

## 五、后续影响

这轮历史 SFT 的价值不是“证明 SFT 全面优于 Base”，而是帮后续路线定边界：

1. SFT 更适合作为格式、协议和工具 grounding 的 warm-up。
2. 复杂偏好选择不应只靠 SFT 模仿 teacher。
3. 后续必须引入 DPO / pairwise preference 来学合法候选之间的选择。
4. 评估报告不能只写 loss，必须写 Base -> SFT -> DPO/Rerank 的规则指标变化。
