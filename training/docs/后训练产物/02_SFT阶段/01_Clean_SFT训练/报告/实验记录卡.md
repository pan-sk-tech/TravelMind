# 实验记录卡：Clean SFT 训练

## 一、阶段目标

Clean SFT 的目标是验证：在固定 PlannerContext 和 prompt 协议后，模型能否通过监督微调学会稳定输出结构化、可验证、可重算的 `TripPlan`。

这轮是 SFT 阶段真正的起点，把模型从“prompt 勉强可用”推进到“协议稳定、预算语义明显改善”。

## 二、训练与配置

主线候选来自 `main_clean` 数据路线，并比较 lr6e-5 / lr8e-5 等配置。后续主线采用 `valloss_lr8e5` 作为当前候选。

关键配置归档：

- `training/docs/后训练产物/02_SFT阶段/01_Clean_SFT训练/配置/sft_qwen25_7b_lora_260509_main_clean_official_lr6e5.yaml`
- `training/docs/后训练产物/02_SFT阶段/01_Clean_SFT训练/配置/sft_qwen25_7b_lora_260509_main_clean_official_lr8e5.yaml`
- `training/docs/后训练产物/02_SFT阶段/01_Clean_SFT训练/配置/sft_qwen25_7b_lora_260509_main_clean_cp2_legacy_b_valloss_lr6e5.yaml`
- `training/docs/后训练产物/02_SFT阶段/01_Clean_SFT训练/配置/sft_qwen25_7b_lora_260509_main_clean_cp2_legacy_b_valloss_lr8e5.yaml`

评估口径：standard 200 条 + hard 300 条，vLLM 推理，temperature=0.2，workers=10。

## 三、关键结果

相对 base，Clean SFT 大幅提升 hard pass、soft pass、预算关系和餐饮尺度。

| split | 指标 | base | lr8e-5 | 变化 |
| --- | --- | ---: | ---: | ---: |
| standard | hardpass | 89.90% | 96.46% | +6.56pp |
| standard | softpass | 5.56% | 61.11% | +55.55pp |
| standard | 重算预算 softpass | 13.13% | 45.45% | +32.32pp |
| standard | 预算关系 | 6.06% | 56.57% | +50.51pp |
| standard | 餐饮尺度 | 35.35% | 73.23% | +37.88pp |
| hard | hardpass | 83.28% | 91.97% | +8.69pp |
| hard | softpass | 3.34% | 46.15% | +42.81pp |
| hard | 预算关系 | 5.69% | 45.15% | +39.46pp |
| hard | 餐饮尺度 | 38.80% | 68.56% | +29.76pp |

## 四、阶段结论

1. SFT 能显著改善协议稳定性和结构化规划习惯。
2. 预算关系、餐饮价格尺度和多样性不是 prompt 能稳定解决的问题，SFT 后明显改善。
3. `lr8e-5` 在 standard 的 hardpass 和 softpass 最好；hard 集 hardpass 略低于旧数据历史强对照，但 softpass 和预算相关指标更好。
4. Clean SFT 的价值是打地基，不是解决最终旅行质量全部问题。

## 五、后续影响

Clean SFT 成为后续 Usage700 Replay、Patch700 诊断、Best-of-N 和 DPO 的基础节点。后续所有实验都需要和它比较，判断新增数据到底是在补强还是偏科。
