# 实验记录卡：Usage700 Replay

## 一、阶段目标

Usage700 Replay 的目标是把真实预算口径修正后的 usage 样本重新喂给模型，观察它能否在不破坏 Clean SFT 主线的情况下，提高预算结构关系、餐饮尺度和 hard split 鲁棒性。

## 二、训练配置

- 起点：上一版 Clean SFT 主线。
- 数据：`main_clean + realbudget usage700 replay`。
- 训练配置：`training/docs/后训练产物/02_SFT阶段/02_Usage700_Replay/配置/sft_qwen25_7b_lora_260511_main_clean_plus_realbudget_usage700_from_lr6e5_lr2e5.yaml`
- 启动脚本：`training/docs/后训练产物/02_SFT阶段/02_Usage700_Replay/配置/run_260511_realbudget_usage_patch_from_lr6e5.sh`
- serve 配置：`training/docs/后训练产物/02_SFT阶段/02_Usage700_Replay/配置/.serve_qwen25_7b_replay_usage700_gpu4.yaml`

## 三、关键结果

相对上一主线 `prev_lr8e5`：

| split | 指标 | prev_lr8e5 | replay_usage700_lr2e5 | 变化 |
| --- | --- | ---: | ---: | ---: |
| standard | hardpass | 96.46% | 95.50% | -0.96pp |
| standard | softpass | 61.11% | 61.00% | -0.11pp |
| standard | 重算预算 softpass | 45.45% | 50.50% | +5.05pp |
| standard | 预算关系 | 56.57% | 81.00% | +24.43pp |
| standard | 餐饮尺度 | 73.23% | 83.00% | +9.77pp |
| hard | hardpass | 91.97% | 97.32% | +5.35pp |
| hard | softpass | 46.15% | 52.51% | +6.36pp |
| hard | 预算关系 | 45.15% | 67.22% | +22.07pp |
| hard | 餐饮尺度 | 68.56% | 79.60% | +11.04pp |

## 四、阶段结论

1. Usage700 Replay 是健康补强，尤其提升 hard split 鲁棒性。
2. 预算结构关系和餐饮价格尺度大幅改善，说明真实预算口径样本有效。
3. standard hardpass 小幅回落，但 softpass 基本持平，整体可以接受。
4. hard 的重算预算贴合从 53.85% 降到 48.83%，说明“实际选择贴预算”仍未解决。

## 五、后续影响

Usage700 Replay 成为后续 Best-of-N 600 的基础模型。它证明了追加 SFT 数据可以定向改善预算语义，但也暴露了 SFT 很难同时优化所有预算子目标。
