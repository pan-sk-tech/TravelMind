# 实验记录卡：Patch700 Only 诊断

## 一、阶段目标

Patch700 Only 的目标不是替代主线，而是诊断：如果只用 realbudget usage patch 样本继续训练，模型会学到什么、会不会偏科。

## 二、训练配置

- 起点：上一版 Clean SFT 主线。
- 数据：`realbudget usage patch700 only`。
- 训练配置：`training/docs/后训练产物/02_SFT阶段/03_Patch700_Only诊断/配置/sft_qwen25_7b_lora_260511_realbudget_usage_patch700_only_from_lr6e5_lr1e5.yaml`
- 启动脚本：`training/docs/后训练产物/02_SFT阶段/03_Patch700_Only诊断/配置/run_260511_realbudget_usage_patch_from_lr6e5.sh`
- serve 配置：`training/docs/后训练产物/02_SFT阶段/03_Patch700_Only诊断/配置/.serve_qwen25_7b_patch700_only_gpu5.yaml`

## 三、关键结果

相对上一主线 `prev_lr8e5`：

| split | 指标 | prev_lr8e5 | patch700_only_lr1e5 | 变化 |
| --- | --- | ---: | ---: | ---: |
| standard | hardpass | 96.46% | 97.50% | +1.04pp |
| standard | softpass | 61.11% | 34.50% | -26.61pp |
| standard | 重算预算 softpass | 45.45% | 30.50% | -14.95pp |
| standard | 预算偏好 | 77.27% | 45.00% | -32.27pp |
| hard | hardpass | 91.97% | 98.33% | +6.36pp |
| hard | softpass | 46.15% | 35.79% | -10.36pp |
| hard | 预算偏好 | 72.58% | 57.86% | -14.72pp |
| hard | 餐饮尺度 | 68.56% | 66.22% | -2.34pp |

## 四、阶段结论

1. Patch700-only 能提升 hardpass，说明它确实补到了硬约束稳定性。
2. 但 softpass、预算偏好和重算预算质量明显回退，不能作为主线。
3. 它适合当“硬约束修补方向”的诊断样本，不适合单独作为综合质量模型。
4. 这轮说明：SFT 数据如果分布过窄，会让模型偏科。

## 五、后续影响

Patch700 Only 没有进入主线，但它帮我们明确了后续数据混比原则：不能只用 patch 样本追硬指标，必须和 replay / best-of-n / 多样性样本混合。
