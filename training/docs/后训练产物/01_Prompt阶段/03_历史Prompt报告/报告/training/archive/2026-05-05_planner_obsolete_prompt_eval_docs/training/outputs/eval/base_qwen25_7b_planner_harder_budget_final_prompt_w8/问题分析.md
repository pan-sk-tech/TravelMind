# V3 Harder Baseline Prompt 对比

本报告对比同一份 300 条 harder eval 在三版 prompt 下的表现。后置校验未介入，全部为模型原始输出规则评估结果。

| 版本 | JSON可解析 | Schema通过 | Hard Pass | 预算加总一致 | 预算档位/偏好 | 景点数 | 三餐完整 | 天气匹配 | 主要错误 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | --- |
| 原 harder prompt | 98.00% | 97.33% | 92.12% | 93.49% | 98.63% | 99.66% | 99.32% | 99.66% | budget_arithmetic_inconsistent=19, json_extract=6, budget_preference_mismatch=4, schema=2, weather_mismatch=1 |
| 错误数字示例 prompt | 99.33% | 99.00% | 84.51% | 85.86% | 98.65% | 99.33% | 99.66% | 99.66% | budget_arithmetic_inconsistent=42, budget_preference_mismatch=4, json_extract=2, weather_mismatch=1, schema=1 |
| 最终无数字示例 prompt | 99.33% | 99.00% | 93.27% | 93.94% | 97.64% | 99.66% | 98.99% | 100.00% | budget_arithmetic_inconsistent=18, budget_preference_mismatch=7, json_extract=2, schema=1 |

## 结论

- `错误数字示例 prompt` 不采用：具体预算数字示例会污染输出，模型会复用示例中的 `total_attractions` 等字段，导致预算一致性下降。
- `最终无数字示例 prompt` 是当前最好 baseline：JSON/schema 更稳，预算加总一致略好于原版，hard_pass 从 92.12% 提升到 93.27%。
- 剩余主要问题仍是 `budget_arithmetic_inconsistent=18` 和 `budget_preference_mismatch=7`。这说明 prompt 可以降低格式风险，但预算闭合/预算贴合仍可能需要输出端确定性重算或 DPO 阶段优化决策偏好。
- 本轮未启用后置校验；4396 vLLM 服务保持运行，便于继续评估。
