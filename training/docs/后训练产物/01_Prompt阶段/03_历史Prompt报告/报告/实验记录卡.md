## 三、关键实验发现

Prompt 阶段的主要发现是：硬约束最强版本和软质量最强版本不是同一个版本。

| 版本 | hardpass | softpass | 说明 |
| --- | ---: | ---: | --- |
| `base_qwen25_7b_v3_harder_baseline_symbolic_w16_rerun` | 88.55% | 8.42% | 硬约束稳定，但餐饮重复严重。 |
| `base_qwen25_7b_v3_harder_ablation_meal_rotation_symbolic_w12` | 77.89% | 43.20% | 软质量最高，但硬约束下降。 |
| `base_qwen25_7b_v3_harder_ablation_meal_grounded_diversity_symbolic_w12` | 80.87% | 39.93% | 餐饮 grounding 和语义较好，但重复仍明显。 |
| `base_qwen25_7b_v3_harder_food_bucket_return_dinner_prompt_w8` | 82.59% | 38.57% | 工程折中参考。 |

这个结果直接决定了后续路线：prompt 可以定协议、定口径、定上下文，但不适合继续承担所有质量优化。

## 四、最终 baseline 决策

| baseline | 用途 | 选择理由 |
| --- | --- | --- |
| `base_qwen25_7b_v3_harder_baseline_symbolic_w16_rerun` | 硬约束 baseline | hardpass 高、schema 稳、预算算术强。 |
| `base_qwen25_7b_v3_harder_ablation_meal_rotation_symbolic_w12` | prompt-only 软质量上限 | softpass 最高，但不能作为唯一线上口径。 |
| `base_qwen25_7b_v3_harder_food_bucket_return_dinner_prompt_w8` | 折中参考 | 可作为工程 prompt 参考，但不是训练主线终点。 |

## 五、阶段结论

1. Prompt 阶段已经接近上限，继续调文案会在 hardpass 和 softpass 之间来回摇摆。
2. route hints、预算 few-shot、self-check 等策略都有局部收益，但都有副作用。
3. 餐饮多样性、预算花得舒服、路线审美和偏好权衡应交给后训练和 rerank。
4. 后续训练报告必须继承 Prompt 阶段定下来的 JSON 协议、PlannerContext 口径和评测指标。
