## 一句话结论

这一版 baseline 的优势是 **格式协议、grounding、天气/日期、酒店晚数、景点数量等工程硬指标非常稳**；短板是 **预算是否花得合理、路线/交通是否现实、餐饮是否具体、软偏好和负向约束是否真正被权衡**。这说明 DPO 阶段很有价值：它不该再教模型写 JSON，而该教模型“选更好的旅行方案”。

### 典型问题示例

#### 偏好/负向约束

- `v3_harder_eval_000007` 景点选择集中在热门经典（武侯祠、熊猫基地、宽窄巷子），完全未覆盖小众展馆和艺术偏好
- `v3_harder_eval_000007` 宽窄巷子属于过度商业化景点，与用户明确回避要求相悖
- `v3_harder_eval_000005` 总预算2900元超出2600元硬约束，预算分项自洽性差

#### 交通/路线/位置

- `v3_harder_eval_000007` 每日更换远离市区的民宿，交通极度不便且与现实脱节
- `v3_harder_eval_000005` 每日三餐完全重复相同餐厅，且餐厅位置与当日景点严重不顺路，缺乏连贯性
- `v3_harder_eval_000005` 包含北京路步行街、广州塔等明显商业化打卡点，违反避开商业化打卡点约束

#### 餐饮/美食

- `v3_harder_eval_000007` 正餐多为虚构的“宽窄巷子特色餐厅”或咖啡店简餐，缺乏真正的特色餐厅安排
- `v3_harder_eval_000005` 每日三餐完全重复相同餐厅，且餐厅位置与当日景点严重不顺路，缺乏连贯性
- `v3_harder_eval_000006` 缺少海洋馆、缺乏本地特色美食体验，未充分满足偏好

#### 酒店/住宿

- `v3_harder_eval_000007` 每日更换远离市区的民宿，交通极度不便且与现实脱节
- `v3_harder_eval_000005` 民宿位于白云区石井，距离市中心主要景点通勤时间约70-90分钟，不符合“不要太偏远”偏好
- `v3_harder_eval_000006` 每日更换酒店且酒店位置偏远，单程交通时间超过60分钟限制，对带幼儿家庭极不现实

#### 预算/费用

- `v3_harder_eval_000007` 预算总额仅2220元，远低于7000元舒适预算，且门票分项计算错误，严重不符预算要求
- `v3_harder_eval_000005` 总预算2900元超出2600元硬约束，预算分项自洽性差
- `v3_harder_eval_000006` 餐饮费用计算严重错误：计划4天每天3顿每顿180元，实际总餐费应为2160元，而非预算中的540元，导致实际总费用超4000元硬约束

#### 工具/事实忠实

- `v3_harder_eval_000007` 正餐多为虚构的“宽窄巷子特色餐厅”或咖啡店简餐，缺乏真正的特色餐厅安排
- `v3_harder_eval_000016` 未满足亲子核心偏好：行程未安排用户明确要求的动物园和海洋馆（最后一天的苏州海洋馆在工具快照中标记为免费且位于吴中区偏远位置），且选用的凯悦酒店并非标注的亲子酒店，缺乏儿童友好设施描述
- `v3_harder_eval_000051` 餐饮描述泛泛，未使用工具候选的特色餐厅，仅标“特色云南菜”，未满足品质用餐偏好

#### 其他

- `v3_harder_eval_000045` 最后一天返程仍安排景点并重复武馆，缺乏合理性；全计划单调重复，缺乏城市漫步和慢节奏的具体体现
- `v3_harder_eval_000019` 第四天重复游览丽江古城, 连续4日缺乏多样性, 整体行程单调
- `v3_harder_eval_000074` 缺少自由活动安排，行程没有体现慢节奏和灵活度

#### 天气/季节适配

- `v3_harder_eval_000002` 景点安排路线严重跳跃（如灵隐寺跨至萧山稻乡漫步街），不考虑实际交通时间和雨天影响，缺乏可执行性
- `v3_harder_eval_000052` 连续雨天（11月6日小雨，7日中雨）未作任何室内调整或备选方案，与天气信息脱节
- `v3_harder_eval_000063` 第一天中雨仍安排户外深圳湾公园，缺乏天气应对，老人体验差

## 可作为简历/汇报的表述

- 构建了包含规则评估与 LLM-as-Judge 的双层评测体系：规则层验证结构化协议与工具 grounding，Judge 层评估真实旅行决策质量。
- V3 baseline 在困难评估集上 hard pass 达到 93%+，说明 prompt 与 PlannerContext 协议已能稳定约束格式和基础事实。
- LLM judge 暴露出明显决策短板：预算合理性 1.59/5、可执行性 1.91/5、综合质量 1.82/5，为 DPO 数据构造提供了明确优化方向。
- 后续 DPO 应优先学习交通路线合理性、餐饮落地性、预算使用质量、软偏好/负向约束权衡，而不是继续优化 JSON 格式。
