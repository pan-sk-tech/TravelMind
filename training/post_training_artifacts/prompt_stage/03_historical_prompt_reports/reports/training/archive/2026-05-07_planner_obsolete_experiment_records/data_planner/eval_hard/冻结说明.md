## 当前结论

base + v3 prompt 在更难评估集上仍然守住了 `hard_pass >= 90%`，说明当前协议和 prompt 已经相当稳。

主要剩余问题：

- 预算算术闭合：`budget.total` 和四个分项加总不一致。
- JSON 数字字段里偶发输出算式，例如 `450 * 4`，导致不是合法 JSON。
- 极少数 schema 类型问题，例如餐饮费用输出小数、字段名缺失。

这些问题更适合通过 prompt 精修、后置修复和硬过滤解决，不适合作为 DPO rejected。
